# Design System — «Sudolabs-inspired» рестайл портфолио

Документ описывает новую дизайн-систему для `mikhail.shytsko.com`, вдохновлённую
[sudolabs.com](https://sudolabs.com/). Это **не копия** — мы извлекаем их принципы
(палитра, типографика, спейсинг, паттерны секций, моушн) и переносим на твой контент.

Статус: **черновик на ревью.** Кода ещё нет — сначала согласуем направление.

---

## 1. Что мы берём у Sudolabs

Их эстетика — «premium enterprise-tech»: высокий контраст, много воздуха,
сдержанный, но смелый красный. Ключевые наблюдения с живого сайта:

| Приём | Как у них | Что переносим |
|---|---|---|
| **Красный hero** | Полноэкранный красный градиент (≈`#e0001a` → тёмно-бордовый/чёрный), белый текст | Hero-секция как «обложка», крупный градиент вместо неоновых частиц |
| **Чередование тем** | Тёмные пурпурно-чёрные секции ↔ светлые почти-белые | Ритм «тёмная / светлая» между блоками вместо одного сплошного тёмного фона |
| **Eyebrow-лейблы** | Мелкий КРАСНЫЙ uppercase моноширинный текст с разрядкой над заголовком (`AI CAPABILITIES`) | Над каждой секцией — мини-лейбл (`EXPERIENCE`, `PROJECTS`…) |
| **Огромные заголовки** | Очень крупный жирный grotesk, 2 строки | Hero и заголовки секций крупнее и жирнее текущих |
| **Нумерация** | `01 / 02 / 03` красным у шагов и аккордеонов | Опыт работы / проекты как нумерованный список |
| **Stat-блоки** | Гигантские цифры (`90%+`, `500+`) + подпись | Блок ключевых метрик (годы опыта, проекты, и т.п.) |
| **Карточки** | Сетка кейсов, чуть приподнятые карточки на тёмном | Карточки проектов и опыта |
| **Аккордеоны** | FAQ и «capabilities» с номером + шеврон | Раскрывающийся опыт работы (уже есть) + FAQ-блок |
| **Hairline-разделители** | Тонкие линии между пунктами списка | Разделители в списках опыта/образования |
| **Кнопки** | Outline-кнопка, uppercase, разрядка (`LET'S TALK`) | CTA «Связаться» в том же стиле |

## 2. Палитра

Твои текущие цвета (`#1a0033` + `#c10000`) **уже близки** к sudolabs — у них тёмные
секции тоже глубокий пурпурно-чёрный, а акцент красный. Так что рестайл — это в
основном уточнение оттенков + добавление светлой темы для чередования.

### Токены (формат HSL — под существующие CSS-переменные в `globals.css`)

| Роль | Hex | HSL | Назначение |
|---|---|---|---|
| `ink-900` (dark bg) | `#16001f` | `280 100% 6%` | Фон тёмных секций (чуть глубже текущего) |
| `ink-800` (card) | `#21082f` | `283 71% 11%` | Карточки/хедер на тёмном |
| `ink-700` (border) | `#3a1d4d` | `274 45% 21%` | Бордеры, hairline на тёмном |
| `red-600` (primary) | `#d10000` | `0 100% 41%` | Основной акцент, ссылки, номера |
| `red-500` (hero) | `#e6001a` | `353 100% 45%` | Яркий край hero-градиента |
| `paper-50` (light bg) | `#f6f5f8` | `260 18% 97%` | Фон светлых секций |
| `paper-100` (card-light) | `#ffffff` | `0 0% 100%` | Карточки на светлом |
| `ink-on-paper` | `#1b1230` | `262 45% 13%` | Текст/заголовки на светлом |
| `muted-on-paper` | `#6b6478` | `265 9% 43%` | Вторичный текст на светлом |
| `muted-on-dark` | `#b9a9c9` | `276 24% 73%` | Вторичный текст на тёмном |

> Решение для ревью: **полностью убрать фиолетовый как «неоновый»** и оставить его
> только как глубокий тёмный фон. Акцент — исключительно красный. Это даёт более
> «дорогой» вид, как у sudolabs. (Альтернатива — сохранить лёгкое фиолетовое свечение;
> скажи, если хочешь компромисс.)

### Hero-градиент
```css
/* радиальный/диагональный красный, как у sudolabs */
background: radial-gradient(120% 120% at 30% 30%, #e6001a 0%, #b00014 35%, #2a0010 75%, #16001f 100%);
```

## 3. Типографика

Sudolabs использует жирный grotesk для заголовков и моноширинный для лейблов.
Бесплатные аналоги под Next.js (`next/font/google`), без лицензионных рисков:

- **Дисплейные заголовки:** `Space Grotesk` *(или `Inter Tight` — более нейтральный)*
- **Body:** `Inter`
- **Eyebrow-лейблы / цифры-моно:** `JetBrains Mono` *(или `Geist Mono`)*

### Шкала

| Токен | Размер (desktop) | Вес | Применение |
|---|---|---|---|
| `display-xl` | `clamp(3rem, 7vw, 6rem)` | 700 | Hero-заголовок |
| `display-lg` | `clamp(2.25rem, 4.5vw, 3.75rem)` | 700 | Заголовки секций |
| `heading` | `1.5rem` | 600 | Карточки, пункты |
| `body` | `1rem` / `1.125rem` | 400 | Основной текст |
| `eyebrow` | `0.75rem`, `letter-spacing: 0.15em`, uppercase, mono | 500 | Лейблы секций (красные) |
| `stat` | `clamp(2.5rem, 5vw, 4.5rem)` | 700 | Большие цифры метрик |

Заголовки — плотный `line-height: 1.05`, отрицательный трекинг `-0.02em`.

## 4. Спейсинг и сетка

- **Контейнер:** `max-w-6xl` (≈1152px), боковые поля `px-6 md:px-8`.
- **Вертикальный ритм секций:** `py-24 md:py-32` (sudolabs очень «воздушный»).
- **Сетка карточек:** 1 → 2 → 3 колонки (`grid md:grid-cols-2 lg:grid-cols-3`, `gap-6`).
- **Радиусы:** `--radius` поднимаем с `0.5rem` до `0.75rem` (мягче, как у их карточек).
- **Тени:** на светлом — мягкие (`shadow-sm`/`shadow-md`); на тёмном — без теней, разделение бордером.

## 5. Паттерны компонентов

1. **Section eyebrow** — `<p class="eyebrow text-red-600">EXPERIENCE</p>` над каждым `display-lg` заголовком.
2. **Numbered list** — крупный красный `01` слева, заголовок + описание справа, hairline-разделитель снизу. Для опыта и «как я работаю».
3. **Stat row** — 3–4 больших цифры (`stat`) с подписями: например «6+ лет опыта», «N проектов», «AI / Solutions».
4. **Project card** — карточка с тегами-категориями (uppercase mono), заголовком, описанием, ссылкой-стрелкой.
5. **Accordion** — номер + заголовок + шеврон; раскрытие через Framer Motion (у тебя уже есть для опыта — приводим к новому стилю).
6. **CTA button** — `outline`, uppercase, `letter-spacing`, hover-заливка красным.
7. **Logo / skill marquee** — горизонтальная бегущая лента (как их клиентские логотипы) → твой стек технологий.
8. **Footer CTA** — крупный заголовок «Давай поговорим» + кнопка, как их `Let's map your AI opportunity`.

---

## 6. Моушн

Sudolabs — сдержанный моушн, без «неона». Предложение:

- **Убрать/смягчить** канвас-частицы (`particles-background.tsx`): либо удалить, либо заменить
  на статичный красный градиент + лёгкий шум. Частицы с курсором — главный «не-premium» элемент сейчас.
- **Reveal on scroll:** staggered fade-up (`opacity 0→1`, `y 16→0`), длительность 0.5s, ease `[0.16,1,0.3,1]`.
- **Hover:** карточки слегка приподнимаются (`-translate-y-1`) + бордер краснеет.
- **Параллакс** оставить только в hero, очень слабый.

## 7. Чередование секций (предлагаемый ритм)

| Секция | Тема |
|---|---|
| Header (sticky) | Тёмная, прозрачная → `ink-800` при скролле |
| Hero / About | **Красный градиент** |
| Stats | Светлая (`paper-50`) |
| Summary | Тёмная (`ink-900`) |
| Experience | Светлая |
| Projects | Тёмная |
| Education | Светлая |
| FAQ *(новое, опционально)* | Тёмная |
| Contact / Footer CTA | Красный градиент → тёмная |

## 8. Изменения в файлах (план реализации, после апрува)

- `app/globals.css` — новые CSS-переменные (палитра выше), радиус, скроллбар под красный/тёмный.
- `tailwind.config.ts` — токены шрифтов, доп. цвета (`paper`, `ink`), утилиты `eyebrow`/`stat`.
- `app/layout.tsx` — подключить шрифты через `next/font` (Space Grotesk + Inter + JetBrains Mono).
- `app/page.tsx` — применить паттерны секций, eyebrow-лейблы, чередование тем, нумерацию.
- `components/particles-background.tsx` — упростить/заменить (см. §6).
- *(опц.)* новый компонент `components/marquee.tsx` для ленты навыков.

## 9. Открытые вопросы для тебя

1. **Фиолетовый:** убрать совсем (только тёмный фон) или оставить лёгкое свечение? *(по умолчанию — убрать)*
2. **Частицы:** удалить, упростить до градиента, или оставить как есть?
3. **Шрифты:** `Space Grotesk` (характерный) или `Inter Tight` (нейтральнее) для заголовков?
4. **FAQ-секция:** добавлять новую (как у них) или не нужно для портфолио?
5. **Контент Stats:** какие 3–4 метрики показывать (годы опыта / проекты / стек)?
